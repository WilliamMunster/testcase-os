# Skill: Jira Sync

Integrate with Jira to pull requirements, create bugs, and synchronize status using `scripts/jira-cli.sh`.

- **Trigger Phrases**: "Pull PRD from PROJ-1234", "Create bug for TC-RPP-001", "Sync PROJ-1234 status", "Link PROJ-1234 and TC-RPP-001"

## Configuration

*   Read Jira instance URL, project keys, and credentials from `_system/config.yaml`.
*   Verify `jira-cli` is installed and authenticated.

## Workflow

1.  **Requirement Sync (Pull PRD)**:
    *   Command: `scripts/jira-cli.sh pull-prd {ticket_id}`.
    *   Action: Pull content from a Jira story/task to be used by `case-design`.

2.  **Bug Creation**:
    *   Command: `scripts/jira-cli.sh create-bug {md_file}`.
    *   Action: From a Markdown Bug report (generated by `case-execute`), create a Jira issue.

3.  **Status Sync**:
    *   Command: `scripts/jira-cli.sh sync-status {ticket_id}`.
    *   Action: Synchronize the status of a Jira ticket back to the local MD card.

4.  **Issue Linking**:
    *   Command: `scripts/jira-cli.sh link-case {ticket_id} {case_id}`.
    *   Action: Link a Jira requirement ticket with one or more test case IDs.

## Mapping

| Input Intent | Script Action |
|---|---|
| "Pull PRD from {ticket}" | `pull-prd {ticket}` |
| "Create Bug for {id} failure" | `create-bug {md_file}` |
| "Sync {ticket} status" | `sync-status {ticket}` |
| "Link {ticket} and {id}" | `link-case {ticket} {id}` |

## Error Handling

*   If script fails, display the error output from `jira-cli.sh`.
*   Ensure the `config.yaml` has the required `jira` section before attempting actions.
